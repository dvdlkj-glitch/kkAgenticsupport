PART 1: SYSTEM GUIDANCE & INTAKE POLICY FOR USERS
--------------------------------------------------------------------------------
This section provides guidance policies that the chat system or support agents 
must communicate to incoming users across all entry points (Web, Telegram, 
Streamlit).

1. File Attachment & Troubleshooting Capability
   * Policy: Actively inform users that they can upload diagnostic materials 
     directly through the interface.
   * Accepted Formats: Issue screenshots (images) or issue log files (.log, .txt)
     generated by the designed application program.
   * Purpose: Drastically reduces troubleshooting turnaround times by providing 
     immediate engineering context.

2. Standard Support Questionnaire Requirements
   * Policy: Users can ask any general or project-specific support questions here.
   * Condition: Before technical diagnostics can begin, the system must enforce
     the collection of critical corporate identifier fields:
     - User's Full Name
     - Company Name
     - Project Name
     - Unique Project Code

3. Special Instructions for STL Trainers
   * Policy: Detect or provide a pathway for users who identify as trainers from STL.
   * Protocol: Require them to provide their full name, and explicitly instruct 
     them to upload the provided text file containing the developmental roadmap 
     or specifications of the project their client wishes to build.


PART 2: AUTO-RESPONDING SCRIPTS & DIALOGUE SEGMENTS
--------------------------------------------------------------------------------
These pre-formatted automated text templates should be triggered during the
relevant stages of user onboarding and issue reporting inside the chat environment.

[SEGMENT A: WELCOME & INITIAL IDENTIFICATION INTAKE]
--------------------------------------------------------------------------------
Trigger: When a new user initializes the chat session.

"Welcome to kkAgentic Support Services. To ensure your inquiry is routed to the 
correct dedicated project team and database, please begin by providing the 
following details:
1. Your Full Name:
2. Your Company Name:
3. The Project Name:
4. Your Designated Project Code:

Please type your reply below to unlock full support diagnostics."


[SEGMENT B: EXPERIENCING ISSUE TRACING]
--------------------------------------------------------------------------------
Trigger: Once the user provides identification details and states they have an issue.

"Thank you for validating your account details. To help us isolate the systemic 
behavior, could you please describe the specific issue you are currently 
experiencing within this project? Please include any error codes, unexpected 
behaviors, or the actions leading up to the incident."


[SEGMENT C: LOG FILE / DIAGNOSTIC UPLOAD PROMPT]
--------------------------------------------------------------------------------
Trigger: When the user explains the issue and needs deeper engineering analysis.

"To help our engineering team run a deep-dive analysis into this behavior, do 
you have an issue log generated by the program? You can easily drag and drop 
the log file or a screenshot directly into this chat window to upload it."


[SEGMENT D: EMAIL INPUT REQUIREMENT (PRE-UPLOAD GATE)]
--------------------------------------------------------------------------------
Trigger: Fired immediately before allowing a file stream attachment to upload.

"Security & Verification Check: Before uploading your log files or screenshots, 
please provide your corporate email address so we can link these diagnostics to 
your ticket and send automated status updates."


[SEGMENT E: EXPECTED RESOLUTION TIMELINE NOTICE]
--------------------------------------------------------------------------------
Trigger: Fired immediately after a log or diagnostic file has been successfully uploaded.

"Diagnostic File Received: Your log file has been securely uploaded to our system. 
Please note that it typically takes a few hours to a full business day for our 
automated pipelines and engineers to fully parse the logs, isolate the issue, 
and get back to you with an actionable solution. Thank you for your patience."


[SEGMENT F: SPECIALIZED FLOW FOR STL TRAINERS & MEMBERS]
--------------------------------------------------------------------------------
Trigger: When the user indicates they are an STL member or trainer.

"Welcome, STL Team Member. To process your client's development request efficiently, 
please reply with:
1. Your Full Name:
2. The Name of the Project you wish to build for your client:

Additionally, please upload your client's requirement specifications file 
named exactly as 'client request.txt' using the file attachment (📎) icon in this 
chat window."
